# handoff-eval.md — Commit Gate

Run in Claude Code when the user pastes the new `session.md` generated by voice chat, BEFORE writing it to the repo (Flow step 4 in `CLAUDE.md`). Never run on fetch — the outgoing `session.md` is handed to voice as-is. Verifies the returned handoff is complete enough for the next session to continue seamlessly.

**Scope:** handoff COMPLETENESS + continuity only. NOT story accuracy, recency, metrics, or factual truth — those are explicitly out of scope. This gate never asks "is the story true," only "can the next session continue without re-explaining."

**Method:** pass/fail per check. Any FAIL → stop, report the gap, do not commit.

**Object under test:** voice outputs only the SESSION HANDOFF half. Checks 4–10 run against voice's pasted block; checks 1–3 run against the assembled `session.md` after Claude Code re-stamps SESSION PERSISTENT from `CLAUDE.md`. If voice rewrote any Persistent content, that is an automatic FAIL.

## Checks
<!-- Ordered top-to-bottom against session.md's layout; each check names the section it protects. -->
1. **Contract present** → ⚓ Operating Contract (re-stamped) — single operating-rules block at top of assembled `session.md`, compiled from CLAUDE.md's `→voice` lines.
2. **Intent + roles** → §Intent (re-stamped) — stated once, clearly.
3. **Stories are pointers** → §Stories — story facts reference `personal-context/`, not restated as new claims.
4. **Date + since-last** → Handoff header — both present; a dateless handoff fails.
5. **Progress table** → §Progress — present; statuses unambiguous; exactly one clear current/next task.
6. **Tangent log** → Tangents line — present (empty is fine).
7. **Active question + rubric inline** → §Current state — the question under work AND the eval rubric for the current task are written in `session.md` (voice can't read the repo).
8. **Concrete next action** → §Next — a doable first move ("tell the real Primavera story"), not "continue working."
9. **Synthetic flagged** → whole handoff — every invented/unverified item is labeled as such. No invented fact presented as real. (Labeling check, not a truth check.)
10. **Critical path clean** → whole handoff — no placeholder TODO blocking the current task; its inputs are all present.

## Output
- All pass → `HANDOFF OK` + 3-line recap → write the pasted content to `session.md` and commit.
- Any fail → `HANDOFF INCOMPLETE` + numbered gaps + the single fix that would make each pass. Do not write or commit.
